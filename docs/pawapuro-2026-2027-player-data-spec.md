# パワフルプロ野球2026-2027 選手データ仕様メモ

このドキュメントは、このリポジトリで扱う選手データの基準仕様です。
今後の UI・JSON スキーマ・バリデーション・インポート/エクスポート実装は、まずこのドキュメントを参照してください。

## 位置づけ

- これは KONAMI の内部仕様書ではありません。
- 公式サイトで確認できる能力表記と、コミュニティの実データ参照ページから観測できる項目を整理した開発用ドキュメントです。
- 事実として確認できた項目と、このツールで採用する実装方針を分けて記述します。

## 公式サイトから確認できる事実

### 1. 能力更新はオンラインで配信される

公式の「オンライン能力更新」説明では、以下が明記されています。

- オンライン接続により、収録済み選手の能力データが更新される。
- 2026年5月中旬頃までの成績を反映した一部選手が対象。
- メンテナンス後、ゲーム内でオンライン接続すると自動更新される。
- 将来のソフトウェアアップデートに合わせて追加更新される予定。

このため、このツールでは将来的に以下を考慮する必要があります。

- データのバージョン管理
- 更新日時の保持
- デフォルト収録値とオンライン更新後値の区別

### 2. 能力値は数値とランク表記が併用される

公式の大谷翔平選手能力紹介では、以下のような表現が使われています。

- パワー S98
- 球速 164km/h
- 特殊能力名の個別表示
- 超特殊能力の存在

ここから確認できること:

- 生の能力値は数値で保持される。
- UI では数値に加えてランク記号が表示されうる。
- 特殊能力には通常特殊能力だけでなく、超特殊能力も存在する。

## Baseless の実データページから確認できる事実

Baseless のページは、実際のゲーム内選手データを人が参照しやすい形式にまとめたものです。内部バイナリ仕様そのものではありませんが、UI/JSON 設計の基礎として非常に有用です。

### 1. 選手基本情報

球団別ページから、少なくとも以下の項目が確認できます。

- 選手名
- 背番号
- 投打
- フォーム種別 + フォーム番号
  - 例: オーバースロー15, スリークォーター16, スタンダード9, オープン118
- 投手/野手の別
- 投手は防御率、野手は打率・本塁打・打点などの参考成績

補足:

- 年齢は Baseless 表示上の必須項目ではない。
- 現在のツールでは age を保持しているが、これはアプリ都合の補助項目であり、ゲーム表示の中核仕様ではない。

### 3. 野手能力の構成

## 現在のアプリ実装との差分

現状アプリでは、次の簡略化があります。

- `hand` が「右投左打」の1文字列
- `position` が守備位置/適性の1文字列
- `traits` がすべて同じ文字列配列
- `breakingBalls` が球種名とレベルだけ
- フォーム情報未対応
- 超特殊能力/固有能力の独立カテゴリ未対応
- 守備位置略号とランク系守備能力の厳密分離未対応

今後の開発では、この簡略化を前提に UI を増やすのではなく、このドキュメントの構造へ寄せる方向で改善する。

## 表示ランクと内部数値の対応ルール

この項目は優先度高で扱う。

重要な前提:

- 公式サイトでは `パワー S98` のように、少なくとも `S` ランク表示の存在が確認できる。
- ただし、2026-2027 版について KONAMI 公式が数値境界を一覧で公開しているわけではない。
- したがって、このツールでは以下を `暫定の表示ルール` として採用する。
- 以後、一覧表示・詳細表示・バリデーションメッセージはこのルールで統一する。

### このツールで採用する暫定表示ルール

野手主要能力、投手のコントロール/スタミナ、および同種の 0〜100 能力値は以下でランク表示する。

| 数値範囲 | 表示ランク |
| --- | --- |
| 90〜100 | S |
| 80〜89 | A |
| 70〜79 | B |
| 60〜69 | C |
| 50〜59 | D |
| 40〜49 | E |
| 1〜39 | F |
| 0 | G |

対象能力:

- ミート
- パワー
- 走力
- 肩力
- 守備力
- 捕球
- コントロール
- スタミナ

対象外:

- 球速
- 弾道
- 変化球レベル
- A〜G そのものを持つランク系特殊能力

### UI 表示ルール

- 一覧では `ランク + 数値` を併記する。
  - 例: `C 63`, `S 98`
- 詳細画面のゲージ・ランクバッジも同じ数値境界を使用する。
- 速度系は `151km/h` のように数値のみを表示する。
- 弾道は `1〜4` の段階表示のみとし、S〜G は付与しない。

### 実装メモ

- 実装上は共通関数を 1 箇所に置き、一覧と詳細で同じ判定を使うこと。
- 将来、2026-2027 固有の正式境界が確認できた場合は、この節を更新し、UI と JSON 変換ロジックを一括で揃える。

## 今後の実装優先順位

### 優先度 高

- traits を ranked / toggle / special に分離
- 守備位置と守備系ランク能力の分離
- 投手適性（先・中・抑）の独立管理
- フォーム種別 + フォーム番号の保持
- 表示ランク（S/A/B...）と内部数値の対応ルールの維持・更新

### 優先度 中

- 超特殊能力の専用カテゴリ化
- 野手/投手/二刀流で表示項目を切替
- 起用法プリセットの整備
- 球種マスタの整備

### 優先度 低

- オンライン能力更新の差分管理
- 公式更新の取り込み補助

## 実装ルール

- 新しい選手データ項目を追加する前に、このファイルへ追記する。
- UI 改修前に、対象項目がこのドキュメント上でどのカテゴリか確認する。
- 仕様が曖昧な項目は「公式確認済み」ではなく「観測ベース」と明記する。
- Baseless の表記を UI 参考にしつつ、公式サイトで確認できた内容を優先する。
